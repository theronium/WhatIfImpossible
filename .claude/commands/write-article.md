## 執筆ルール

- CLAUDE.md に定義されたセクション構成に従う
- frontmatter（title / id / category / tags / date）を必ず含める
  - id は既存記事を確認して次の番号を割り当てる（`docs/` フォルダを確認すること）
  - date は今日の日付
- 日本語で執筆する
- 「実現不可能性の根拠」セクションは物理的・技術的・論理的の3観点から論じる
- 数式は極力使わない。1記事につき最大1つまで。概念は言葉で説明することを優先する
- 論理フローは Mermaid の flowchart で図示する
- 最後に関連しそうなカテゴリや概念を「関連記事」に列挙する（まだ存在しなくてもよい）

## 出力

1. `docs/<category>/<id>.md` に記事を保存する。
2. `docs/README.md` のインデックスを更新する。
   - 該当カテゴリのテーブルに1行追加（ID・タイトル・タグ・日付）
   - 末尾の「記事数: N」をインクリメントする
3. `review-article` スキルを呼び出し、保存したファイルパスを引数として渡してレビュー・修正を行う。
4. 保存したファイルパス・記事タイトル・レビュー結果（修正した項目、問題なしの項目）を報告する。
5. **git commit は行わない。** ユーザーが明示的に指示した場合のみコミットする。
