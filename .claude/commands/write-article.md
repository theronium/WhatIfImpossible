## 執筆ルール

- CLAUDE.md に定義されたセクション構成に従う
- frontmatter（title / id / category / tags / date）を必ず含める
  - id は `node glossary/scripts/next-id.js` を実行して次の番号を確認する
  - date は今日の日付
- 日本語で執筆する
- 「実現不可能性の根拠」セクションは物理的・技術的・論理的の3観点から論じる
- 数式は極力使わない。1記事につき最大1つまで。概念は言葉で説明することを優先する
- 論理フローは Mermaid の flowchart で図示する
- 最後に関連しそうなカテゴリや概念を「関連記事」に列挙する（まだ存在しなくてもよい）

## 出力

1. `docs/<category>/<id>.md` に記事を保存する。
2. `docs/README.md` のインデックスを更新する。
   - 該当カテゴリのテーブルに1行追加（ID・タイトル・タグ・日付）
   - 末尾の「記事数: N」をインクリメントする
3. 技術ツリーに追加すべきノードがあれば更新する。
   - 技術ツリーはインデックス（`docs/notes/tech_tree.md`）とブランチファイル（`docs/notes/tech_tree_<name>.md`）に分割されている
   - 記事の概念が既存ブランチに属する場合は、対応する `docs/notes/tech_tree_<name>.md` にノードを追記する
   - 新しい系統が必要な場合は `docs/notes/tech_tree_<name>.md` を新規作成し、インデックスのブランチ一覧テーブルにも行を追加する
   - どのブランチに属するか判断できない場合はユーザーに確認する
4. `review-article` スキルを呼び出し、保存したファイルパスを引数として渡してレビュー・修正を行う。
5. 保存したファイルパス・記事タイトル・レビュー結果（修正した項目、問題なしの項目）を報告する。
6. **git commit は行わない。** ユーザーが明示的に指示した場合のみコミットする。
