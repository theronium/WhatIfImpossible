## レビュー手順

1. ファイルを Read ツールで読み込む
2. 以下のチェックリストに従って問題を洗い出す
3. 修正可能な問題は Edit ツールで直接修正する（術語の置換・注記の追加など）
4. 構造の根本的な欠如（セクション不足・Mermaid がない）はユーザーに報告して判断を仰ぐ
5. 修正した項目・変更なしの項目・要確認の項目を一覧で報告する

## チェックリスト

### A. 構造
- frontmatter に title / id / category / tags / date が揃っているか
- 「実現不可能性の根拠」に物理的・技術的・論理的の 3 観点があるか
- Mermaid の flowchart（図解セクション）があるか
- 「関連記事」セクションがあるか
- 数式は 1 記事 1 つまでになっているか

### B. 術語
- 「パラ粒子」が使われていたら「パランティ粒子（g161）」に修正する
- WIIM独自概念（コーラ粒子・アンキロン・レトロン・ネゴトン・グラビトーペイク・パランティ粒子・ノーファペン）は初出時に gXXX または wiim_XXX の参照があるか
- 「仮想粒子」（量子揺らぎで生まれる専門用語）を WIIM独自の架空粒子やタキオン等の仮説粒子に誤用していないか
  - 架空・仮説粒子には「思考実験上の粒子」「仮説上の粒子」「理論的粒子」を使う

### C. 物理的正確さ
- 余剰次元にエネルギー・エントロピーを転嫁する場合、余剰次元側の代償・収支を明示しているか
- カシミール効果の説明で「仮想粒子の生成・増幅」を正とするか確認する。使う場合は「比喩的表現であり、正確には真空モード密度の変化」と注記があるか
- フーリエ変換等の解析手法に動的計量（一般相対論）では線形近似範囲でのみ有効という限定があるか（宇宙全体スケールや重力波スペクトル解析の場合のみ）
- 「時間加速」など誤解を招く表現には固有時の視点からの補足があるか
- エネルギー「取り出し」と「変換」を混同していないか（カシミール効果・熱力学サイクルなど）

### D. 論理的整合性
- 余剰次元を「問題を消す」魔法の解決策として使っていないか（先送りか完全回避かを区別する）
- 「スケール大→問題深刻化」という前提と矛盾するスケール設定をしていないか（矛盾がある場合は「意図的な最困難ケース」として明記する）
- 観測装置や記録システムが自己参照的矛盾に陥る場合（装置が観測対象に含まれる等）、その限界に言及しているか

- 「言及漏れ・説明不足」は注記・補足文の追加で対処する（Edit ツール）
